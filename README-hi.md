
# रिएक्ट ऐप बनाएं के साथ शुरुआत करना
यह प्रोजेक्ट [क्रिएट रिएक्ट ऐप](https://github.com/facebook/create-react-app) के साथ बूटस्ट्रैप किया गया था].

## उपलब्ध स्क्रिप्ट

इस प्रोजेक्ट निर्देशिका में, आप चला सकते हैं:

### `npm start`

ऐप को डेवलपमेंट मोड में चलाता है।\
इसे ब्राउज़र में देखने के लिए [http://localhost:3000](http://localhost:3000) खोलें।

यदि आप संपादन करते हैं तो पृष्ठ पुनः लोड होगा।\ आपको कंसोल में कोई भी लिंट त्रुटियाँ भी दिखाई देंगी।

### `npm test`

इंटरेक्टिव वॉच मोड में टेस्ट रनर लॉन्च करता है।\ अधिक जानकारी के लिए [चल रहे परीक्षण] (https://facebook.github.io/create-react-app/docs/running-tests) के बारे में अनुभाग देखें।

### `npm run build`

'बिल्ड' फ़ोल्डर में उत्पादन के लिए ऐप बनाता है।\
यह रिएक्ट को प्रोडक्शन मोड में सही ढंग से बंडल करता है और बेहतरीन प्रदर्शन के लिए बिल्ड को ऑप्टिमाइज़ करता है।

बिल्ड को छोटा किया गया है और फ़ाइल नामों में हैश शामिल है।\
आपका ऐप परिनियोजित करने के लिए तैयार है!

अधिक जानकारी के लिए [तैनाती] (https://facebook.github.io/create-react-app/docs/deployment) के बारे में अनुभाग देखें।

### `npm run eject`

**नोट: यह एकतरफा ऑपरेशन है। एक बार जब आप 'इजेक्ट' कर लेते हैं, तो आप वापस नहीं जा सकते!**

यदि आप बिल्ड टूल और कॉन्फ़िगरेशन विकल्पों से संतुष्ट नहीं हैं, तो आप किसी भी समय `इजेक्ट` कर सकते हैं। यह कमांड आपके प्रोजेक्ट से सिंगल बिल्ड डिपेंडेंसी को हटा देगा।

इसके बजाय, यह सभी कॉन्फ़िगरेशन फ़ाइलों और संक्रमणीय निर्भरता (वेबपैक, बैबेल, ESLint, आदि) को सीधे आपके प्रोजेक्ट में कॉपी करेगा ताकि आप उन पर पूर्ण नियंत्रण रख सकें। `इजेक्ट` को छोड़कर सभी कमांड अभी भी काम करेंगे, लेकिन वे कॉपी की गई स्क्रिप्ट को इंगित करेंगे ताकि आप उन्हें बदल सकें। इस समय आप अपने दम पर हैं।

आपको कभी भी `इजेक्ट` का उपयोग नहीं करना है। क्यूरेटेड फीचर सेट छोटे और मध्यम परिनियोजन के लिए उपयुक्त है, और आपको इस सुविधा का उपयोग करने के लिए बाध्य महसूस नहीं करना चाहिए। हालाँकि हम समझते हैं कि यह उपकरण उपयोगी नहीं होगा यदि आप इसके लिए तैयार होने पर इसे अनुकूलित नहीं कर सकते।
